# 検索品質評価ガイドライン・TL;DR

このページでは、Googleが契約している検索品質評価者向けに厳密に記述された情報を避け、できるだけ簡潔にするよう努めています。

## 検索品質評価ガイドラインの概要

Googleの検索結果内で、ページは2つのことに基づいてランク付けされます。

1. 情報の**品質**、および
2. **ユーザーの意図をどの程度満たしているか**

ランク付けしたいクエリを考えると、ページは何パーセントのユーザーを満足させますか？

## ページ・Webサイトの理解

### Webページ

特定のカテゴリでページを上位にランク付けする場合は、何よりもまず**ユーザーの支援**（競合他社よりも優れている）に焦点を合わせます。

::: tip ページをランク付けするには、役立つ目的または有益な目的が必要です。
 役立つまたは有益なページの目的は次のとおりです（ただし、これらに限定されません）。

- トピックに関する情報の共有。
- 個人情報または社会情報の共有。
- 写真、ビデオ、またはその他の形式のメディアの共有。
- 意見や見解を表明する。
- 面白いユーザー。
- 製品またはサービスの販売。
- ユーザーが他のユーザーが回答できるように質問を投稿できるようにします。
- ユーザーがファイルを共有したり、ソフトウェアをダウンロードしたりできるようにします。

:::

::: warning YMYLページ（金銭・人生に関するページ）
ユーザーの将来の幸福、健康、経済的安定、または安全に影響を与える可能性のあるページ（YMYLページ）は、より厳密に判断されます。

YMYLページの例：

- ショッピングまたは金融取引のページ
- 財務情報ページ
- 医療情報ページ
- 法的情報ページ
- 情報に通じた市民を持つために重要なニュース記事または公開/公式情報ページ
- 子供の養子縁組
- 自動車の安全情報
- など
:::

#### Webページの一部

Googleからのいくつかの定義：

> **MC(メインコンテンツ)**
> : **ページがその目的を達成するのに直接役立つ**ページ上のコンテンツ（製品ページの製品詳細、レシピページの成分/測定/方向など）。
>
> **SC（補助的コンテンツ）**
> : MCまたは広告とは**見なされない**が、**ユーザーエクスペリエンスにプラスまたはマイナスの影響を与える**コンテンツ（ナビゲーションリンク、関連製品、スパムコメントなど）。
>
> **広告・収益化**
> : **ページの収益化**を目的として表示されるコンテンツやリンク。

### Webサイトの理解

Googleは、サブドメインとルートドメインが別々のエンティティとして扱われる可能性が高い時期を記録しています（**強調**が追加されました）。

> ...サブドメインのトップページは、ドメインのトップページではなく、ルートページと見なされます。どうして？**ロゴをクリックすると、ユーザーはサブドメインのトップページに移動します**。さらに、サブドメインのトップページには、サブドメインのWebサイトに関する情報があります。**ドメインのホームページでサブドメインに関する<span style="text-decoration：underline">特定の</span>情報を見つけるのは非常に困難です**。

Googleは、信頼できるさまざまな独立した情報源（[独立行政国民生活センター](http://www.kokusen.go.jp/ncac_index.html)などのサイト）に基づいて、Webサイトの評判を判断します。

::: tip 信頼を得る。
特にニュース/調査/情報ベースのページでは、誰（人/会社）がWebサイトの責任者であり、誰（人/会社）がコンテンツを作成したかをGoogleができるだけ簡単に見つけられるようにします。

一般に、これは全体としてより信頼できるWebサイトを意味します。
:::  

::: tip 連絡先情報とポリシーを含めます。
ビジネスの種類に応じて、Googleはさまざまな種類と量の連絡先情報とポリシーを想定しています。**この情報を提供することは、お金を扱うすべてのWebサイト（店舗、銀行、クレジットカード会社など）にとって非常に重要です。**
:::

#### 連絡先情報の提案

- ショッピングサイト
  - オフィスの住所
  - 電子メールアドレス
  - 電話番号
  - 支払いポリシー
  - 交換ポリシー
  - 返品規則
- ユーモア/エンターテインメントのウェブサイト
  - オフィスの住所
  - 電子メールアドレス
- 個人ブログ
  - メールアドレス/連絡先なし

### ウェブサイトとかMC（メインコンテンツ）の作成者の評判

::: tip 良い評判を維持します。
あなたの会社が満足している顧客からのレビューを求めており、否定的なレビューに応答していることを確認してください。
:::

会社（またはあなた）がオンラインで悪い評判を持っている場合、グーグルは最初に知ることになるでしょう、そしてあなたのサイトは結果として苦しむことができます。この情報は、以下を含むがこれらに限定されない多くのソースから収集および分析されます。

- ニュース記事
- ウィキペディアの記事
- ブログ投稿
- 雑誌記事
- フォーラムディスカッション
- 独立した組織からの評価
- カスタマーレビュー

<div class="note">

中小企業の場合、オンラインで評判がなくても大丈夫です。サイトに悪影響を与えることはないです。

</div>

## Page Quality評価スケール

::: danger 次のいずれかの特性を持つWebページは、ランクが低くなります。

- 有益な目的はありません
- ユーザーを支援しようとはしません
- 憎しみを広める
- 害を及ぼす
- 誤った情報が含まれています
:::

ページの品質は、主に次の5つの要素によって決まります。

1. ページの目的
2. **E**xpertise（専門知識）、**A**uthoritativeness（権威）、および**T**rustworthiness（信頼性）（E‑A‑T）
3. ページの**メインコンテンツ**の質と量
4. **メインコンテンツ**の責任者である会社/個人に関する情報
5. **メインコンテンツ**の責任者である会社/個人の評判

有益な目的を持つページの場合、E‑A‑Tは次のとおりです。

- メインコンテンツの**作成者の専門知識**
- メインコンテンツの**作成者**、**メインコンテンツ自体**、および**ウェブサイト**の**信頼性**。
- メインコンテンツの**作成者**の信頼性**、**メインコンテンツ自体**、および**ウェブサイト**。

### 専門知識、権威、信頼を呼び起こす方法

E‑A‑Tは、Webサイトのトピックに基づいて異なる方法で決定されます。ページおよびサイトタイプに基づくガイダンスについては、[このページにアクセスしてください](quality-characteristics.html)。

## 高品質なページ

::: tip 既知の専門家によって作成されたコンテンツは上位にランク付けされます。
あなたが専門家であるか、専門家を雇うかにかかわらず、コンテンツを作成した専門家に帰属させるようにしてください。

ページに属性がない場合は、会社全体がコンテンツを作成したことを意味します。
:::

高品質のページには、次の特徴があります。

- 高度な専門知識、権威、信頼性（E‑A‑T）
  - コンセンサスが存在する場合、情報は専門家のコンセンサスによってサポートされる必要があります
- **説明的または役立つタイトル**を含む、満足のいく量の高品質MC
  - ページの**E‑A‑T**は、**MCの品質に大きく依存します**
  - コンテンツの作成に多大な時間、労力、専門知識、または才能/スキルが費やされた
  - 情報サイトは、事実上正確で、明確に記述され、包括的である必要があります
  - ショッピングサイトは、簡単な閲覧と簡単な購入を可能にする必要があります
- 満足のいくウェブサイト情報および/またはウェブサイトの責任者に関する情報
  - そしてeコマースサイトの満足のいくカスタマーサービス情報
- コンテンツを作成した人または会社に対する肯定的な評判

## 最高品質なページ

最高品質のページには、次の特性の少なくとも1つが必要です。

- **非常に**高い専門知識、権威、信頼性（E‑A‑T）
  - 正式な専門知識は、医療、財務、または法律上のアドバイスなどのトピックにとって重要です
  - ユーモアやレシピなどのトピックでは、日常の専門知識が重要です
  - 料理の専門家ページは、プロのシェフのWebサイトのページ（正式な専門知識）の場合もあれば、人気の料理動画をYouTubeに投稿する人（日常の専門知識）の場合もあります。
- **例外的な**量の高品質または最高品質のMC
  - 品質基準は、**ページの目的**と**コンテンツの種類**によって異なります。
- コンテンツを作成した個人または企業に対する**例外的に**肯定的な評判
  - 非常に肯定的な評判は、多くの場合、ページのトピックに関する既知の専門家または専門家協会からの権威ある賞または推奨に基づいています
  - ユーモアやレシピなど、正式な専門知識を必要としないトピックは、人気、ユーザーエンゲージメント、ユーザーレビューを通じて並外れた評判を示すことができます

## 低品質なページ

低品質のページには、次の特性の少なくとも1つがあります。

- 低E‑A‑T
  - ウェブサイト/作成者からの主題に関する不十分な専門知識
  - コンテンツの件名がウェブサイトのテーマと一致していません
  - 安全でない（http）トランザクションページ
- 低品質のMC
  - コンテンツに費やされる労力/時間が少ない
  - MCのタイトルが誇張されている（不正確になっている）
  - 多くの文法エラー
  - 多くの句読点エラー
- MCの量が少ない
- 広告または補足コンテンツが気を散らす
  - インタースティシャルページ
  - 広告/ポップアップを閉じるのが難しい
  - 広告はNSFW
- MCのウェブサイト/作成者に関する情報量が少ない（匿名性が不要な場合）
  - カスタマーサービス情報なし
  - 連絡先情報なし
  - 匿名/不明なユーザーによって作成されたYMYLコンテンツ
- MCのウェブサイト/作成者に対するやや否定的な評判

## 最低品質なページ

::: danger 害を与えないで。
ユーザーに**害を及ぼす可能性のある**ページは、Google内で上位にランク付けされません。
:::

最低品質のページには、次の特性の少なくとも1つがあります。

- 憎悪を広める可能性のあるページ
  - 人々のグループに対する憎悪や暴力を助長する
- 潜在的に有害なページ
  - 自傷行為や他人への危害を助長する
  - 悪意のあるページ（詐欺、フィッシング、マルウェアのダウンロードなど）
  - 非常に否定的または悪意のある評判
- ユーザーに誤解を与える可能性のあるページ
  - 明らかに不正確なコンテンツ
  - 確立された専門家のコンセンサスと矛盾するYMYLコンテンツ
  - 暴かれたまたは根拠のない陰謀説
- ユーザーを欺く可能性のあるページ
  - 欺瞞的な意図、たとえば、ユーザーを欺くために作成されたWebサイト
  - 欺瞞的なデザイン、例：MCを装った広告
- 目的のページの欠如
  - MCなし
  - 意味不明なMC
  - MCの目的を特定できません
- 目的を達成できないページ
  - 最低のE‑A‑T
  - MCなし/ほとんどまたは最低品質のMC
  - コピーされたMCまたは自動生成されたMC
  - 障害物またはアクセスできないMC
  - MCのウェブサイトまたは作成者に関する不十分な情報
  - メンテナンスされていないWebサイト、およびハッキング、改ざん、またはスパムされたページ

## 中品質なページ

中品質のページは、次のいずれかに分類されます。

- **何も悪いことはないですが、特別なことは何もないです**
  このページはその目的を達成しますが、高品質または低品質の評価に値するものではありません。
- **強力な高品質評価特性と混合**
  ページまたはWebサイトには、いくつかの軽度の低品質特性に加えて、強力な高品質評価特性があります。

## Page Quality評価の要点

::: tip 高い専門知識、権限、および信頼（E‑A‑T）を維持します。
ページは、WebサイトとMCの作成者の両方の評判とE‑A‑Tに基づいて判断されます。**これは、ページがYMYLトピックをカバーしている場合に特に重要です。**
:::

## モバイルのユーザー・クエリ・検索結果を理解する

::: tip モバイルユーザー向けのデザイン。
あなたがあなたのサイトをデザインしているとき、モバイルユーザーについて考えてください。

- フォームはモバイルで簡単に入力できますか？
  - つまり、入力フィールドとボタンは大きくて選択しやすいですか？
- モバイルでサイトをナビゲートするのは簡単ですか？
  - つまり、グローバルナビゲーションはページのどこからでもアクセスできますか？コンテンツ内のリンクは大きくてタップしやすいですか？
- モバイルユーザーのクエリにできるだけ早く回答/解決していますか？
  - つまり、ユーザーのクエリに対する答えはスクロールせずに見える範囲ですか？インタースティシャルや広告でユーザーの邪魔をしていませんか？
:::

::: tip すべてのユーザークエリには目標があります。
特定のクエリに最もよく答えるページが最高にランク付けされます。次の点に注意してください。

- 特定のクエリの検索意図は時間の経過とともに変化する可能性があります（例：2007年と現在の<span class="query">iphone</span>の検索）
  - 古いページは時間の経過とともに関連性が低くなります
- クエリには複数の意味があります（例：<span class="query">apple</span>）
  - クエリの解釈が支配的である場合（この場合は「AppleInc。」）、その解釈に関連するページが最高ランクになります
:::

## Needs Metスケールを使用した品質評価

検索結果は、ユーザーのクエリをどの程度満たしているかに基づいてランク付けされます。

### Fully Meets評価

<div class="note">

ユーザーのニーズを完全に満たすことが理想的ですが、これらのタイプのクエリとそのキーワードは、多くの場合、ターゲットを絞った最適化を必要としません。代わりに、[高度な会議](#highly-meets評価)クエリに焦点を当てることをお勧めします。

</div>

検索結果がクエリに完全に一致する一意のカテゴリである特別なカテゴリ。たとえば、日本の誰かが<span class="query">ブルースクレイジャパン</span>を検索している場合、彼らは間違いなく[https://bruceclay.jpn.com/](https://bruceclay.jpn.com/)。

このタイプの結果は、**Webサイト**クエリと**Know Simple**クエリ専用にほぼ排他的に予約されています。

::: tip ユーザーのクエリを完全に満たすページの特性：

- ページで「Fully Meet」できるのは、具体的で明確で明確なクエリのみです。
- ユーザーが探しているものをすぐに取得または使用するには、ページに最小限の労力が必要です。
- 満足するために別の結果を必要とするユーザーはいません。
:::

### Highly Meets評価

検索結果は、**ほとんど**のモバイルユーザーに役立ちます。他の一部のユーザーは、より多くの結果を見たいと思うかもしれません。

::: tip ユーザーのクエリを非常に満たすページの特性：

-コンテンツは[高品質](../qrg/page-quality-rating-guideline/4-high-quality-pages.html#高品質ページの特徴)
-内容は正確です
-作成者またはビジネスは、このトピックに関して[権威があります](../qrg/page-quality-rating-guideline/3-overall-page-quality-rating.html#専門性・権威性・信頼性-e‐a‐t-の詳細)。
-コンテンツは面白い
-コンテンツは新しくて最近のものです（例：トピックに関する最新ニュース）
:::

### Moderately Meets評価

あなたのページは、多くのユーザーにとって役立つ場合、**または**一部のユーザーにとって非常に役立つ場合に、クエリを適度に満たします（適度に満たします）。一部のユーザーは、より多くの結果を見たいと思うでしょう。
::: warning ユーザーのクエリを適度に満たすページの特性：

- コンテンツはそれほど包括的ではありませんが、低品質ではありません
- コンテンツは最新ではありませんが、古くはありません
- サイトまたは作成者はあまり信頼できません
:::

### Slightly Meets評価

少数のユーザーにのみ役立ちます。 多くまたはほとんどのユーザーは、より多くの結果を見たいと思うでしょう。

::: warning ユーザーのクエリをわずかに満たすだけのページの特性：

-このページは[珍しい解釈](../qrg/needs-met-rating-guideline/16-rating-queries-with-multiple-interpretations-and-intents.html)のクエリ
-ページは[低品質](../qrg/page-quality-rating-guideline/6-low-quality-pages.html)です。
-情報が古くなっています
-コンテンツが不正確
-コンテンツが具体的すぎる
-コンテンツが広すぎる
-一般の人々にはニッチすぎる
-タイトルは[誤解を招くまたは誇張されています](../qrg/page-quality-rating-guideline/6-low-quality-pages.html#低品質のメインコンテンツ)
:::

### Fails to Meet評価

これらのページは、ユーザーのニーズを完全に満たす/満たすことができません。 すべてまたはほとんどすべてのユーザーが、より多くの結果を見たいと思うでしょう。

::: danger ユーザーのクエリを満たさないページの特性：

-内容が事実上正しくない
-ページの品質が非常に低い
-コンテンツが非常に古くなっています
-ページをモバイルデバイスで使用することはほぼ不可能です
-ページはユーザーに害を及ぼす可能性があります
   -悪意のあるダウンロード
   -フィッシングページ
-ウェブサイトが別のブランドになりすます
-コンテンツは[ユーザーを欺く](../qrg/page-quality-rating-guideline/7-lowest-quality-pages.html#欺瞞的なページの目的)ことを試みます
   -偽のレビュー
   -フェイクニュース
-不正確な[YMYL情報](../qrg/page-quality-rating-guideline/2-understanding-webpages-and-websites.html#ymylページ-金銭・人生に関するページ)を提供します
   -不正確な医療情報
   -不正確な税務情報
   -不正確なビザ/グリーンカード情報
-ポルノ広告のある非ポルノページ
-確立された科学的、医学的、または歴史的コンセンサスと矛盾するページ
:::

### Page QualityとNeeds Met評価の関係

ページに[有用な目的がない]（#ページ・webサイトの理解）がある場合、そのページはランク付けされません。

<div class="note">

[ウェブサイトのクエリ](../qrg/understanding-mobile-user-needs/12-understanding-mobile-users-queries-and-results.html#ウェブサイトクエリ)は、上記のルールの例外です。 ただし、これらのタイプのクエリは、新規顧客を対象としていません。

</div>

### 複数の解釈・意図があるクエリの評価

クエリの主要な解釈は常に最高にランク付けされます。

たとえば、米国で<span class="query">[halo](https://www.google.com/search?q=halo)</span>を検索している**ほとんど**のユーザーは、ビデオゲームに関する情報を探しています。 それがクエリの背後にある最も一般的な意図であるため、ビデオゲームに関するページは、[光学現象](https://en.wikipedia.org/wiki/Halo_(optical_phenomenon))または[頭上に浮かぶリング](https://en.wikipedia.org/wiki/Halo_(religious_iconography))に関するページよりも上位にランク付けされます。

### 情報の鮮度とNeeds Met評価

ニュース速報、定期的なイベント、現在の情報、または製品に関するクエリの場合、情報が新しいほど良いです。

### 訪れるクエリの評価

Googleは、「近く」をどれだけ近づけるかを決定する際に、エンティティのタイプ（検索対象）を考慮に入れます。 「近く」とは、ユーザーが何を探しているかによって、まったく異なる意味を持つ場合があります。例として、以下をテストします。

- [近いガソリンスタンド](https://www.google.com/search?q=近くのガソリンスタンド)
- [近い映画館](https://www.google.com/search?q=近い映画館)
- [近い美術館](https://www.google.com/search?q=近い美術館)
- [近い国立公園](https://www.google.com/search?q=近い国立公園)
